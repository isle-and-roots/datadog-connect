# ブラウザ自動取得ガイド

## 仕組み

Playwright（Node.jsのブラウザ自動化ライブラリ）を使い、各サービスの管理画面をブラウザで開きます。ユーザーが手動でログインした後、必要な情報を自動的に取得します。

## 対応サービス

| サービス | 取得内容 |
|---------|---------|
| **Datadog** | API Key, Application Key（新規作成） |
| **AWS** | Account ID |
| **GCP** | Project ID（IAM設定ページから取得） |
| **Azure** | Subscription ID |
| **Xserver** | VPS一覧, ファイアウォール自動設定 |

## セキュリティ

- ブラウザは**常に画面表示**（ヘッドレスにしない）— 何が起きているか見える
- ログインは**ユーザーが手動**で行う — ツールがパスワードを知ることはない
- SSO / 二要素認証 もそのまま対応
- 取得した値は**バリデーション**してからAPIに渡す（不正値は手動入力にフォールバック）
- ブラウザセッションは try/finally で確実にクリーンアップ

## 初回セットアップ

Playwrightのブラウザ（Chromium）が未ダウンロードの場合、自動で確認されます:

```
🌐 ブラウザ自動取得を使うには、初回のみブラウザのダウンロードが必要です。
? ダウンロードしますか？ (約100MB) (Y/n)
```

## ブラウザが使えない場合

- Playwrightが未インストール → 手動入力にフォールバック
- ブラウザ取得に失敗 → 手動入力にフォールバック
- ユーザーが「手動入力」を選択 → 従来の対話形式で入力

**3段階フォールバック**: 環境変数 → ブラウザ → 手動入力
