セットアップ手順

5ステップで
完全セットアップ

コマンドを実行して、画面の指示に従うだけ。難しい設定は一切不要です。

1

準備

まず Node.js（v20以上） をインストールします。 nodejs.org からダウンロードできます。
インストール後、ターミナルで node -v と入力して v20.x.x 以上が表示されればOKです。

次に、ツールをダウンロードします。

ターミナル
git clone https://github.com/isle-and-roots/datadog-connect.git cd datadog-connect npm install
2

セットアップ開始

以下のコマンドを実行すると、対話形式のウィザードが起動します。

ターミナル
npx datadog-connect setup

起動すると、最初に認証方法を選択します。ブラウザで自動取得がおすすめです。認証に失敗しても3回まで再入力可能。API Key の確認方法も案内されます。

datadog-connect 
$ npx datadog-connect setup

完了

セットアップが完了すると、完了レポートが表示されます。

  • 🎉 自動設定済み — Datadogの管理画面にダッシュボードやモニターが作成されています
  • 📄 手動手順書 — ~/.datadog-connect/output/ にIAMロール作成など手動手順が出力されます
  • ↩️ やり直したい場合 — npx datadog-connect rollback で作成したリソースを削除できます

主要コマンド

コマンド 説明
npx datadog-connect setup セットアップウィザードを起動します。初回はここから。
npx datadog-connect resume 前回途中で止まったセットアップを、失敗した箇所から再開します。

よくある質問

FAQ

困ったときに参照してください。

Datadogにログインしたときのブラウザのアドレスバーを確認してください。

US1：app.datadoghq.com と表示されている場合
AP1：ap1.datadoghq.com と表示されている場合（日本リージョン）
EU1：app.datadoghq.eu と表示されている場合

ほとんどの場合は US1 または AP1 のいずれかです。ログインURLで判別してください。
ブラウザが自動起動しない場合や操作がうまくいかない場合は、手動入力に切り替えてください。取得した値は自動でバリデーション。不正な値は手動入力に切り替わります。

手動入力では以下の情報が必要です：
  ・Datadog API Key（Organization Settings → API Keys）
  ・Datadog Application Key（Organization Settings → Application Keys）

これらの情報はDatadogの管理画面から取得できます。
npx datadog-connect resume を実行すると、失敗したモジュールだけを再実行できます。最初からやり直す必要はありません。

完了済みのモジュールはスキップされるので、中断した箇所から続けて実行できます。
npx datadog-connect rollback を実行すると、このツールで作成したダッシュボード・モニターなどのリソースをまとめて削除できます。

ロールバックは作成したリソースのリストを参照して実行されるため、手動で作成したものは削除されません。安心して実行してください。
npx datadog-connect mcp を実行することで、Claude Code（AIアシスタント）とDatadogを接続できます。

接続後は Claude Code で次のように話しかけるだけで Datadog を操作できます：
  ・「Datadogの直近のアラートを確認して」
  ・「CPU使用率が高いホストを教えて」
  ・「本番環境のエラーログを検索して」
  ・「新しいモニターを作成して」
お使いのDatadogプランで利用できない機能は自動的にスキップされます。

例えば、CSPM（クラウドセキュリティ姿勢管理）やSIEM（セキュリティ情報イベント管理）はDatadog Enterprise プランが必要な場合があります。

機能を有効化したい場合は、Datadogのプランのアップグレードをご検討ください。
セキュリティ

安全に使うための
設計

Datadog Connectはセキュリティを最優先に設計されています。

🔐
認証情報は保存されません
APIキーやApplicationキーはセッションファイルに保存されません。操作が完了すると認証情報はメモリから消去されます。
🛡️
WAFはmonitorモードで作成
ASM（アプリケーションセキュリティ）のWAFルールは「monitorモード」（検出のみ、ブロックしない）で作成されます。急な通信断のリスクがありません。
🔒
出力ファイルは暗号化権限で保存
手順書などの出力ファイルはアクセス権限 0o600（所有者のみ読み書き可）で保存されます。他のユーザーから参照できません。
🌐
ブラウザ操作は常に画面表示
ブラウザ自動操作はヘッドレスモード（非表示）を使用しません。何が行われているかを常に画面で確認できます。
